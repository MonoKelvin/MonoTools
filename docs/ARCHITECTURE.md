## 2. 分层职责

### 2.1 Layer 1: 用户层 (User Layer)

**职责**: 接收用户输入，发起操作请求。

| 入口 | 实现 | 说明 |
|------|------|------|
| 全局快捷键 | `windows-rs` `RegisterHotKey` | 系统级热键，即使软件无焦点也能捕获 |
| 系统托盘 | Tauri `TrayIcon` API | 右键菜单：显示/隐藏、设置、退出 |
| CLI | `clap` crate | `monotools.exe --search "query"`、`--workspace save` 等 |

### 2.2 Layer 2: 前端层 (Frontend Layer)

**职责**: 渲染 UI，处理交互，通过 IPC 与后端通信。

**核心组件**:
- `SearchBox.vue`: 搜索框主容器，接收输入，展示结果列表
- `ResultList.vue`: 虚拟滚动结果列表，支持键盘导航
- `SettingsPanel.vue`: 设置面板，分插件配置、主题、快捷键
- `PluginViewLoader.vue`: 动态加载插件提供的 Vue 组件

**前端禁止直接调用系统 API**，所有操作通过 Tauri IPC 转发。

### 2.3 Layer 3: IPC 桥接层

**职责**: 安全、高效的前后端通信。

| 机制 | 方向 | 场景 |
|------|------|------|
| `invoke()` | 前端 → 后端 | 搜索请求、保存设置、执行命令 |
| `emit()` | 后端 → 前端 | 索引更新通知、插件加载事件、主题切换 |
| `listen()` | 双向 | 实时搜索建议、进度推送 |

**安全**: Tauri 2.0 Capability 模型，前端只能调用白名单内的指令。

### 2.4 Layer 4: Rust 后端核心

#### 4.1 命令总线 (Command Bus)

**设计目标**: 所有功能（包括插件功能）统一通过命令总线执行，CLI 和 GUI 共用同一套逻辑。

```
输入: "search:files query=report.docx limit=50"
    │
    ▼
[CommandParser] → 拆分为 Command { namespace, action, args }
    │
    ▼
[CommandRouter] → 根据 namespace 路由到对应模块
    │
    ▼
[CommandExecutor] → 执行并返回 Result<CommandResponse>
    │
    ▼
序列化为 JSON → 返回前端或 CLI stdout
```

#### 4.2 核心服务 (Core Services)

| 服务 | 职责 | 关键 crate |
|------|------|-----------|
| `PluginManager` | 插件生命周期、依赖解析、权限控制 | 自研 |
| `SearchEngine` | 查询解析、多源结果聚合、排序 | `rusqlite`, 自研 |
| `WorkspaceManager` | 快照序列化/反序列化、恢复调度 | `serde`, `windows-rs` |

#### 4.3 系统服务 (System Services)

| 服务 | 职责 | 关键 API |
|------|------|---------|
| `UsnIndexer` | NTFS USN Journal 读取、MFT 遍历 | `DeviceIoControl`, `FSCTL_ENUM_USN_DATA` |
| `HotkeyService` | 全局快捷键注册/注销 | `RegisterHotKey`, `UnregisterHotKey` |
| `WindowManager` | 窗口枚举、位置状态获取/设置 | `EnumWindows`, `GetWindowPlacement`, `SetWindowPlacement` |
| `ProcessEnum` | 进程枚举、命令行参数获取 | `EnumProcesses`, WMI `Win32_Process` |
| `FileWatcher` | 文件系统变更监控 | `ReadDirectoryChangesW` 或 `notify` crate |
| `AppScanner` | 扫描开始菜单、注册表获取已安装应用 | 注册表遍历、Shell 链接解析 |
| `TrayManager` | 系统托盘管理 | Tauri `TrayIcon` |
| `ConfigStore` | 配置持久化 | `rusqlite` + JSON 字段 |

**依赖规则**:
- 上层可调用下层，下层不可反向依赖
- 同级模块通过事件总线或消息通道通信，禁止直接引用
- 系统服务层对上层暴露 `trait` 接口，具体实现可替换

## 4. 进程与线程模型

### 4.1 主进程 (Tauri Main Process)

- **单进程架构**: Tauri 默认单进程 + WebView2 渲染进程
- **主线程**: 运行 Tauri 事件循环、窗口管理
- **Tokio 运行时**: 多线程异步池，处理 IO 密集型任务（搜索、文件监控）
- **阻塞线程池**: 用于 USN Journal 索引构建等 CPU 密集型任务

### 4.2 线程分配

| 任务 | 线程/运行时 | 说明 |
|------|------------|------|
| 主事件循环 | 主线程 | Tauri 要求 |
| HTTP/IPC 处理 | Tokio RT | 默认多线程 |
| USN 索引构建 | 专用线程池 (`rayon`) | 并行处理多卷 |
| 文件监控 | Tokio 任务 | 异步 IO |
| 搜索查询 | Tokio RT | 快速响应 |
| 插件脚本执行 | 隔离线程 | 防止阻塞主循环 |

### 4.3 并发安全

- 核心服务状态使用 `Arc<RwLock<T>>` 或 `Arc<Mutex<T>>` 共享
- 插件管理器使用 `RwLock` 允许多读单写
- 数据库连接使用 `r2d2` 连接池

## 6. 启动流程

```
[系统启动 / 用户手动启动]
    │
    ▼
加载 monotools.exe
    │
    ├── 1. 初始化日志 (tracing)
    ├── 2. 加载配置 (ConfigStore)
    ├── 3. 初始化数据库连接 (SQLite)
    ├── 4. 注册全局快捷键 (HotkeyService)
    ├── 5. 初始化系统托盘 (TrayManager)
    ├── 6. 启动 Tokio 运行时
    ├── 7. 加载内置插件 (builtin:*)
    │      ├── 注册命令到 Command Bus
    │      ├── 注册搜索提供者到 SearchEngine
    │      └── 注册主题到 ThemeEngine
    ├── 8. 加载用户插件 (plugins/ 目录)
    │      ├── 解析 plugin.json
    │      ├── 校验权限 (Capability)
    │      └── 激活插件 (调用 activate hook)
    ├── 9. 启动文件索引服务 (USN Indexer)
    │      └── 后台线程构建索引
    ├── 10. 启动文件监控 (FileWatcher)
    ├── 11. 隐藏主窗口，进入静默模式
    │
    ▼
[等待用户输入...]
```

## 7. 唤出/隐藏流程

```
用户按下 Alt + Space
    │
    ▼
[HotkeyService] 捕获 WM_HOTKEY 消息
    │
    ▼
检查窗口状态:
    ├── 若隐藏 → 唤出流程
    │      ├── 1. 计算屏幕中央坐标
    │      ├── 2. 设置窗口位置 (居中，固定尺寸 800x520)
    │      ├── 3. 清空搜索框，聚焦输入
    │      ├── 4. 显示窗口 (无任务栏图标，置顶)
    │      ├── 5. 播放进入动画 (scale + opacity)
    │      └── 6. 预加载最近使用 / 热门结果
    │
    └── 若显示 → 隐藏流程
           ├── 1. 播放退出动画
           ├── 2. 清空搜索结果缓存
           ├── 3. 隐藏窗口 (不销毁，保持 WebView 状态)
           └── 4. 释放焦点给之前的前台窗口
```

**窗口属性**:
- 尺寸: 800px × 520px（默认，可配置）
- 位置: 屏幕水平居中，垂直偏上 1/3 处
- 样式: 无边框、无标题栏、无任务栏按钮、置顶
- 背景: 透明或毛玻璃（由主题插件控制）
- 失焦行为: 自动隐藏（可配置）
